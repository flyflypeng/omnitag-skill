### 📖 Introduction

**OmniTag Recommendation Skill** is an intelligent tagging assistant designed for personal knowledge management. It automatically generates standardized, hierarchical tag combinations for your notes, articles, or thoughts based on your **personalized tag configuration** and the **PARA method**.

This skill helps you build an orderly knowledge base by applying the classic **"PARA + Topic + Type + Meta" (3+1)** structure.

### 📦 Quick Start

You can install this skill directly from GitHub using `npx skills`:

```bash
npx skills add https://github.com/flyflypeng/omnitag-skill --skill omnitag-recommendation -g -y
```

> ⏳ **Coming Soon**: Easier installation via Clawhub will be supported soon.

### ✨ Features

- **Standardized Tagging**: Generates tags following the `PARA + Topic + Type + Meta` structure.
- **Personalized Configuration**: Adapts to your own tag system defined in `~/.omnitag/omni-tags.yaml`.
- **Intelligent Content Analysis**:
  - Analyzes plain text directly.
  - Automatically extracts main content from URLs (removing ads/nav bars) using `markdown.new` or `Jina Reader`.
  - Supports special handling for WeChat Official Account articles via `wechat-article-to-markdown`.
- **Tag Growth**: Provides scripts to automatically sync new tags generated by the AI back to your configuration file.

### ⚙️ Setup

**⚠️ Prerequisite**: This skill relies on your custom tag configuration file. Please complete the following initialization before use:

1.  **Create Python Environment (Recommended using `uv`)**:
    To avoid dependency conflicts, it is recommended to use `uv` to create an independent virtual environment.
    ```bash
    # 1. Create virtual environment (recommended in ~/.omnitag directory)
    uv venv
    
    # 2. Activate virtual environment
    source .venv/bin/activate
    
    # 3. Install dependencies
    uv pip install pyyaml
    ```

2.  **Prepare Configuration File**: Ensure `~/.omnitag/omni-tags.yaml` exists locally.
    - If not, create it manually using the [YAML Template](SKILL.md#配置文件模板-omnitagomni-tagsyaml).
    - **Note**: Without this file, the skill cannot load your personalized tag system.

### 🚀 Usage

#### 1. Tag Recommendation
Simply provide the content or URL you want to tag. The skill will analyze it and recommend tags based on your configuration.

#### 2. URL Content Extraction
For URLs, the skill uses a companion script to extract content:
```bash
python scripts/url_to_markdown.py "https://example.com/article"
```
- **Primary Engine**: [markdown.new](https://markdown.new/)
- **Fallback Engine**: [Jina Reader](https://jina.ai/reader/)

#### 3. Tag Growth
If the skill recommends new tags (e.g., a new Project or Topic), use the update script to sync them:
```bash
python scripts/update_tags.py "#PARA/Project/NewProject" "#Topic/NewTopic"
```

### 🏷️ Core Rules (3+1 Structure)

| Dimension | Description | Constraint | Example |
| :--- | :--- | :--- | :--- |
| **PARA** | Core Classification (Project/Area/Resource/Archive/Inbox) | **Mandatory & Unique** | `#PARA/Project`, `#Inbox` |
| **Topic** | Content Theme (Domain) | **Mandatory (1-4)** | `#Topic/Agent`, `#Topic/LLM` |
| **Type** | Note Type (Form) | **Mandatory (1-3)** | `#Type/Note`, `#Type/Insight` |
| **Meta** | Metadata (Attribute/Status) | Optional | `#Meta/Daily`, `#Meta/Important` |
